LLM Council — Strategic Brainstorm

2026-04-04 · Landscape Skill: Strategic Intelligence Layer · 5 advisors, 5 peer reviews

What strategic intelligence layer should we add to the /landscape skill to transform it from a data dump into an actionable competitive intelligence briefing?

The Recommendation

Adopt the decision-anchored framework (4 executive decisions), the computation-first architecture (deterministic Python, not LLM inference), and scenario analysis as the differentiator.

Core principle: "Never let the LLM substitute for the math." Compute indices deterministically. Claude narrates the results.

The One Thing to Do First

Write strategic_scoring.py. Take existing landscape CSVs, compute a Competitive Position Index (weighted sum of: pipeline breadth, phase advancement, mechanism diversity, attrition rate), produce a single ranked table. No LLM involvement. Just the math. If the numbers don't pass a smell test against known competitive landscapes, the entire strategic layer is built on sand.

Implementation Roadmap

Phase 1: Strategic Scoring (~3 days)

strategic_scoring.py — Competitive Position Index per company, mechanism crowding score, momentum vector. Pure Python. Output: ranked table with 5 columns + data freshness watermark.

Phase 2: Opportunity Matrix (~1 week)

opportunity_matrix.py — Mechanism-vs-phase heatmap with attrition-adjusted white space. Parameterize by therapeutic area (oncology preset, rare disease preset). Integrate with /mechanism-explorer.

Phase 3: Strategic Narrative (~1 week)

strategic_narrative.py — Structured LLM prompts producing 2-page briefing anchored to 4 executive decisions: enter/expand, partner/acquire, double down/cut, differentiate. Includes scenario analysis: recompute rankings with top drug removed to expose fragility. Shorter than current report, not longer.

Key Analytical Products (from all advisors)

1. Competitive Position Index (Contrarian + Executor)

Phase-weighted pipeline count + mechanism diversity (Herfindahl index) + deal velocity + trial intensity. Ranks companies deterministically. The backbone everything else builds on.

2. Mechanism Crowding vs Attrition Matrix (Contrarian + First Principles)

Active drugs per mechanism vs attrited drugs per mechanism. Identifies: Graveyards (high attrition, still crowded — beta amyloid), Emerging bets (low attrition, growing), White space (low density + low attrition = unexploited opportunity).

3. Company 2x2 Matrix (Outsider)

Pipeline Strength vs Momentum. Four quadrants: Leaders (strong + accelerating), Fading Giants (strong + decelerating), Rising Challengers (weak + accelerating), Struggling (weak + decelerating).

4. Scenario Analysis (Outsider — differentiator)

"What if the lead drug fails Phase 3?" Remove top drug, recompute company rankings. Shows how fragile or robust each company's position is. Turns a report into a decision tool.

5. Attrition Risk Scorecard (Expansionist)

Per-mechanism historical attrition rate by phase. For any drug, its probability of reaching market given mechanism and current phase. Actuarial logic applied to drug development.

Blind Spots the Council Caught

Data Freshness

Stale data + strategic layer = false precision. Need a "data as of" watermark and staleness check on every output.

Configurability by Therapeutic Area

Oncology (800 drugs) vs rare disease (12 drugs) need different scoring models. Weights must be parameterized, not hardcoded.

No Validation Mechanism

How do you know the CPI is predictive? Need backtesting against known competitive dynamics before trusting the scores.

No Iteration Model

A one-shot briefing is a report. True intelligence needs drill-down: "explain this score," "zoom into Phase 3 oncology."

Cross-Skill Integration

/drug-comparison, /mechanism-explorer, /indication-deep-dive already exist. The strategic layer should compose with them, not rebuild parallel pipelines.

Advisor Responses

The Contrarian — "Build the math first"

Don't bolt on LLM narrative — you'll get hallucinated strategy. Build computational analytics: Competitive Position Index, Crowding vs Attrition matrix per mechanism. The analytical backbone must be deterministic Python. Build the math first. Let the LLM narrate the math second. Never let the LLM substitute for the math.

The First Principles Thinker — "Anchor to 4 decisions"

A pharma executive makes 4 decisions: enter/expand, partner/acquire, double down/cut, differentiate. Build: mechanism-phase density matrix, attrition-adjusted opportunity, momentum vectors, concentration risk. Every piece of intelligence maps to a decision or gets cut. Output should be SHORTER, not longer.

The Expansionist — "6 analytical products"

White Space Map, Company Strategy Profiles (archetypes: Broad Platform, Focused Bet, Fast Follower, Acquirer), Mechanism Lifecycle Analysis (emergence/hype/shakeout/maturation), Deal Intelligence (leading indicator), Trial Velocity Dashboard, Attrition Risk Scorecard. Each is a Python script consuming existing CSVs.

The Outsider — "2x2 matrix + scenario analysis"

Finance analogy: scoring model → ranked lists → narrative. Three axes: Pipeline Strength, Momentum, Risk. Plot companies on 2x2 (Leaders/Fading Giants/Rising Challengers/Struggling). Add scenario analysis: what if lead drug fails? Recompute rankings. Two-page output + appendix.

The Executor — "3 scripts, 2 days"

strategic_scoring.py (~200 lines), opportunity_matrix.py (~150 lines), strategic_narrative.py (~100 lines). Add 3 lines to SKILL.md. Tables become appendix, scored tables + narrative lead. Two days for prototype. Ship scoring first, add narrative second.

Peer Review Highlights
  • Strongest: Contrarian's "computation first" principle + Outsider's scenario analysis
  • Data freshness: Stale snapshots + strategic layer = false precision
  • Configurability: Oncology vs rare disease need different scoring weights
  • Validation: No backtesting proposed — how do you know CPI works?
  • Iteration: One-shot report vs drill-down intelligence
