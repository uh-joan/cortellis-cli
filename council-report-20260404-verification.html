LLM Council Report — Verification

2026-04-04 · Pipeline Fix Verification · Follow-up to initial assessment

Have the issues identified in the first council assessment of the /pipeline skill been adequately addressed by the fixes?

The Recommendation

Adequate for controlled, internal deployment with a technically literate operator. Not adequate for broad rollout or any context where the manifest is treated as a correctness guarantee rather than an existence check.

Ship with these conditions:

  • Verify manifest invocation is in the default execution path, not optional. If it requires explicit calling, reclassify it as a diagnostic tool.
  • Fix the denominator before quality metrics are surfaced downstream. A metric certified by the manifest but computed on the wrong base is worse than no metric.
  • Treat SKILL.md as a liability until it accurately describes actual behavior. The gap creates false confidence.

The One Thing to Do First

Audit whether the manifest is invoked automatically in every normal pipeline run or requires explicit operator action. If not automatic, it's a debugging utility counted as a safety fix. That determination changes the entire assessment.

Context: The manifest calls are documented in SKILL.md, which Claude Code reads as orchestration instructions. Since Claude Code IS the runner, the manifest is invoked if and only if Claude follows SKILL.md. This is a documentation-as-code architecture.

Where the Council Agrees

The manifest is a real improvement. Every advisor acknowledges it establishes an audit trail where none existed before. The six fixes addressed operational problems that were genuinely present. Nobody argues the project is worse than before.

The council also agrees that two open issues — credential lifecycle (#7) and wrong denominator (#12) — carry active corruption risk rather than just technical debt.

Where the Council Clashes

Is "adequate" = trust restored or trust complete? The First Principles Thinker draws the line: the manifest answers "did the run produce data?" and that's enough to ship responsibly. The Contrarian and Outsider reject that — a manifest that passes zero-byte CSVs is theater, and SKILL.md now describes a system more capable than the code.

The decisive clash: Is the 98% denominator a corruption risk or a roadmap item? The First Principles Thinker calls it enrichment. Peer review 2 pushes back: if the denominator is wrong, the manifest certifies a corrupted quality metric. That's a false negative in the audit system itself.

Blind Spots the Council Caught

Manifest Invocation Path

Is the manifest actually called in normal pipeline execution, or does it require deliberate invocation? If opt-in, it provides no protection in default runs. In this architecture, Claude Code reads SKILL.md as orchestration — so the manifest IS invoked if Claude follows the instructions.

Manifest-Metrics Coupling

If the denominator feeding quality metrics is wrong, the manifest certifies a corrupted value. "87% coverage" computed against the wrong total, signed off by the manifest, is worse than no metric. Nobody modeled this failure cascade.

No Triage Logic for Open Issues

The six open issues have no assigned severity, blast radius, or owner. "Remaining issues are roadmap items" is deferral dressed as categorization.

Advisor Responses

The Contrarian

The fixes are superficial patches on a fundamentally broken pipeline. The manifest validates file existence but not contents — zero-byte CSV passes. Raising 50→150 just kicks the can. The optional total_trials arg means callers can ignore it. Stderr warnings nobody monitors isn't error handling. Issues 7, 9-12 remain open. The 98% denominator alone invalidates quality metrics.

The First Principles Thinker

The framing of "6 of 12 remain open" is misleading. The actual problem was silent data degradation. The manifest fixes that — it's the architectural primitive. Issues 9-12 are analytical enrichment, not trust failures. The condition "can an operator tell if a run produced complete data?" is now answered yes. Adequate trust repair. Remaining are roadmap items.

The Expansionist

The manifest is a floor, not a ceiling. The 20+ other skills should be wired in. Credential lifecycle and wrong denominator will silently corrupt trust — fix before shipping broadly. Attrition signal is the biggest adjacent opportunity. Fixes addressed operational hygiene; untapped value is cross-skill orchestration and longitudinal trending.

The Outsider

The fixes address symptoms not the system. Each is the minimum intervention. The manifest is bureaucracy not correctness — if it fails, garbage flows through. SKILL.md now describes a system more trustworthy than reality. Issues 7-12 untouched. Original assumptions still baked in.
