## Answer (2026-04-05)

Derived analytical outputs generated by the `/landscape` skill — strategic
briefings, scenario libraries, CPI rankings, mechanism crowding indices,
report markdown — **may be shared with third parties** (external BD partners,
investors, consultants, clients, public benchmarks) subject to the following
conditions:

1. **Attribute** Cortellis / Clarivate as the underlying data source on any
   externally-shared artifact.
2. **Do not redistribute raw Cortellis data.** Anything under `raw/` (CSV
   pulls, `*.meta.json` API responses, company/trial/deal tables fetched
   directly from the Cortellis API) remains governed by the subscriber's
   Cortellis / Clarivate agreement and must stay internal.
3. **Derived = transformed.** Rankings, scores, narrative briefings, and
   scenario analyses qualify as derived outputs. A CSV that is a verbatim
   or lightly-filtered copy of a Cortellis API response does NOT qualify
   and must stay internal.
4. Commercial use of derived outputs is permitted within the terms of the
   active Cortellis subscription.

Because the derived-output question is answered permissively, the code in
this repository (CLI, recipes, skills, schemas, docs, tests) is licensed
**MIT** — see repository `LICENSE`.

## Why It Matters

Every output pathway the landscape skill might monetize or operationalize
redistributes derived Cortellis data:

1. **BD decision artifacts** — Strategic briefings shared with external
   partners, divestment targets, or co-development candidates.
2. **Open-source benchmarks** — Publishing anonymized CPI rankings or
   mechanism crowding indices to GitHub or research repos.
3. **Compliance-gated reports** — IR decks, investor presentations, or
   regulatory submissions referencing Cortellis-derived scores.
4. **API endpoints** — Third-party integrations (e.g., partner dashboards)
   consuming `/landscape` outputs.
