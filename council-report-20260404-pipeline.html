2026-04-04 · Pipeline Skill Completeness Assessment · 5 advisors, 5 peer reviews

Is the /pipeline skill in the Cortellis CLI complete enough for production use? What gaps exist?

The Recommendation

Not production-ready. Ship as beta with an integrity layer added first.

The skill works for known large pharma where resolution is reliable and the user can sanity-check the output. It is not ready for any context where a user will act on the output without manual verification.

The gaps are not feature gaps. They are trust gaps. An operator cannot currently tell if a pipeline run produced complete data or silently degraded data. That single fact disqualifies it from production.

The Expansionist's platform vision and the First Principles Thinker's question framing are both correct about the ceiling — but the floor has to be fixed first. A pipeline that sometimes produces false confidence is worse than no pipeline.

The One Thing to Do First

Add a run manifest. After every pipeline execution, write a structured JSON file recording each step, its exit code, the row count of every CSV produced, the timestamp, and any truncation flags.

Make report generation read this manifest and refuse to render if any upstream step produced zero rows or a non-zero exit. This single change converts silent failures into loud ones, surfaces the filename mismatch, the empty-CSV-from-timeout, and credential failure modes simultaneously — without requiring changes to the underlying recipe scripts.

Where the Council Agrees

Every advisor found the same underlying failure mode: silent, undetectable errors. The Executor named a specific bug (filename mismatch producing false positives in other.csv). The Outsider found the same class of bug independently (empty CSV from API timeout generates a complete-looking report). The Contrarian named the category (50-hit cap with no warning).

This is not a collection of separate issues. It is one architectural failure: the pipeline has no integrity layer. It assumes every step succeeds and reports confidently regardless of whether it did.

The second point of convergence: the resolution logic has an invisible accuracy cliff. 98% on major pharma sounds good. Mid-size biotechs — the more interesting targets for competitive intelligence — fail at ~20%. Three reviewers flagged this. No one disagreed.

Where the Council Clashes

Is this a reporting tool or an intelligence platform? The Expansionist and First Principles Thinker argue the pipeline answers the wrong question — it reports what exists instead of signaling momentum, vulnerability, and competitive position. The Executor and Outsider implicitly reject this: the thing doesn't correctly report what exists yet, so platform ambitions are premature.

The Executor is right on sequencing. You cannot build a competitive intelligence hub on a foundation that silently truncates drug lists and generates reports from empty CSVs. The First Principles Thinker's critique is correct but premature.

Is the AI-reads-markdown architecture a problem? The Outsider calls it a tutorial, not a production system. No other advisor challenged this. The honest answer: Claude Code reading SKILL.md and running bash scripts is unconventional but not disqualifying. What the Outsider is actually pointing at is the absence of orchestration guarantees — a real problem dressed in architectural clothing.

Blind Spots the Council Caught

Credential Lifecycle

Flagged by every reviewer independently. A multi-step pipeline that silently fails at step 4 due to a 401 or expired token produces a partial report with no indication of where it broke. No advisor addressed this.

SKILL.md Drift

SKILL.md may not stay synchronized with actual scripts as the codebase evolves. Since Claude Code uses SKILL.md as the orchestration source, drift means Claude runs wrong commands — and the user has no way to know. Maintenance risk that compounds over time.

False-Alarm Truncation

report_generator.py flags truncation at ≥50 rows even when pagination legitimately returned 51+ real drugs. The warning cries wolf, users learn to ignore it, and real truncation becomes invisible. This makes the Contrarian's concern worse than stated.

Advisor Responses

The Contrarian

The skill has a structural deception problem that no pagination fix will solve.

The 50-hits-per-phase cap is a lie waiting to happen. Pfizer has 300+ pipeline assets. Pagination caps at 150 max per phase with zero warning when you hit it. A user runs /pipeline on Pfizer and gets a "complete" report missing 40% of their oncology portfolio. Silent incompleteness.

Company resolution 98% success rate on 50 major pharma is the wrong denominator. Users will run this on mid-size biotechs, spinouts, recently-renamed companies, subsidiaries. That's where the 2% becomes 20%.

Phase deduplication logic is fragile at the edges. Drugs with ambiguous or missing indication data get silently dropped or misattributed.

Zero data freshness signals. No timestamps on fetched data. A user can't tell if they're looking at today's pipeline or cached state from last week.

The flagship problem: This skill presents itself as definitive but is actually best-effort-with-silent-failures. That's the production gap.

The First Principles Thinker

The question being asked is "is this complete enough?" The question that should be asked is "complete enough for what decision?"

Pipeline intelligence is not a static snapshot problem. It is a signal-detection problem. Someone running this skill is trying to answer: What is this company betting on? Where are they vulnerable? Where are they winning? The current architecture answers none of these explicitly — it answers "what exists" and leaves the inference to the user.

No change detection. A pipeline without delta is a photograph, not intelligence. What moved from Phase 2 to Phase 3 last quarter? Without temporal comparison, users cannot detect momentum.

No competitive anchoring. A company's pipeline in isolation is meaningless. Phase 3 for what indication, against what competitors?

No attrition signal. Attrition is explicitly excluded from catch_missing_drugs. But attrition IS the signal for a company under pressure.

The Expansionist

The pipeline skill is scratching the surface of what's possible.

The untapped API surface is a goldmine. Regulatory timelines + pipeline = approval window predictions. Conference data + pipeline = BD intent signals. Press releases + deals + pipeline = narrative arc reconstruction.

Cross-skill integration. Every drug in that pipeline table could be an entry point into drug-profile, head-to-head, deal-deep-dive. The pipeline should be the hub of a connected intelligence graph.

Temporal dimension completely missing. Where was this pipeline 6 months ago? What moved? What got dropped? Patent watch + pipeline delta = the actual competitive signal pharma analysts pay for.

Partner identification. Mechanism explorer + partnership network + pipeline assets = automated BD target list generation. That's not a feature, that's a product.

The Outsider

The "flagship" framing is doing a lot of work. The skill is a recipe card, not a production system.

Specific code issues: report_generator.py hardcodes 50 as truncation threshold but pagination goes to 150 — they don't talk to each other. trials_summary caps at 15 indications with no warning. Total trials count uses len(rows) not @totalResults.

The 2% resolution failure rate has no documented fallback. What does a user do when their company isn't in the 98%?

No way to know if a run succeeded. No checksums on CSVs, no validation that each step produced output. If ci_drugs_to_csv.py produces 0 rows from an API timeout, the report generates silently with empty tables.

The Executor

Three production blockers ranked by damage:

1. Filename mismatch (step 7): catch_missing_drugs.py reads "phase1.csv"/"discovery.csv" but pipeline writes "phase1_ci.csv"/"discovery_ci.csv". Result: other.csv is bloated with false positives. No error thrown, no warning.

2. No error handling on API failures: fetch_all_drugs has a bare except: break. A transient 429 or network blip silently truncates the drug list.

3. /tmp/pipeline/ is not persistent across reboots or container restarts.

What works fine: Pagination logic, dedup approach, report generator structure, 4-strategy company resolution.

Monday morning fix: In catch_missing_drugs.py line 46, change "phase1.csv" to "phase1_ci.csv" and "discovery.csv" to "discovery_ci.csv". Two-line fix that unblocks production.

Peer Review Highlights

Peer Review Summary (5 reviews)

Strongest advisor (unanimous): The Executor — only advisor with falsifiable, immediately fixable, verifiable bug findings.

Weakest advisor (consensus): The Expansionist — zero actionable production guidance, evangelized cross-skill integration before addressing silent correctness bugs.

Blind spots surfaced by reviewers:

  • Authentication/credential lifecycle: No advisor addressed what happens when API credentials expire mid-run. Flagged by 4 of 5 reviewers independently.
  • SKILL.md drift: Load-bearing documentation may desynchronize from actual scripts over time.
  • False-alarm truncation: report_generator.py flags ≥50 rows as truncated even when pagination legitimately fetched more, training users to ignore the warning.
  • Error surfacing: No structured run manifest — operators cannot distinguish a clean run from a broken one.
