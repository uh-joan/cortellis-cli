Is the /head-to-head skill complete enough for production use? What gaps exist?

The Recommendation

Do not ship today. Ship after a focused half-day of P0 work.

A skill called "head-to-head" that only fetches launched drugs from a single phase endpoint produces silently incomplete comparisons for any company pair where the pipeline matters — which is most of them. That is not a missing feature; it is a correctness defect.

The One Thing to Do First

Expand the fetch logic to call all four phase endpoints (Launched, Phase 3, Phase 2, Phase 1), then add a completeness indicator when any endpoint hits the pagination ceiling. This converts "silently wrong answers" to "substantially correct answers with known limitations."

Blind Spots the Council Caught

Asymmetric Data Quality

When Company A has rich Cortellis data and Company B does not, the comparison is structurally unfair with no user warning.

Overlap Matrix Edge Cases

Correctness degrades when drug-indication mappings are ambiguous or multi-mapped. Analytical correctness of the core output is unexamined.

No Test Strategy

Production readiness discussed without defining how to verify it. No test coverage, no test data, no smoke test.

Error UX on Unhappy Paths

What the user sees when company resolution fails, credentials expire, or API times out is untested.

Advisor Responses

The Contrarian

Closer to ready than the gap list suggests. Launched-only is fine for BD executives comparing commercial portfolios. 20-drug cap covers 80% of mid-cap comparisons. Core value (overlap matrices, side-by-side, ASCII charts) is underappreciated. Ship with single fix (raise limit to 100) + scope label. Pipeline comparison is v2.

The First Principles Thinker

Must answer "who has stronger position now and future?" Three legs: commercial strength, pipeline depth, strategic positioning. Current skill covers commercial + strategic, completely fails on pipeline depth — a missing leg of a three-legged stool. Without multi-phase pipeline comparison, answers the wrong question.

The Expansionist

Natural integration hub. Needs: all-phase pipeline, shared indications battle map, partnership network integration, trial counts, patent expiry windows. Emit structured JSON alongside markdown. Skill is 60% complete — missing 40% is the most strategically valuable part.

The Outsider

"Head-to-head" in pharma means clinical trial comparison — naming creates expectation confusion. 20 drugs is a fetch limit not display limit — analysis itself incomplete. Need fetch-complete-then-summarize + data completeness indicator.

The Executor

P0 (1-2hr): 4 phase calls per company, fix load_json, credential check, resolution error handling. P1 (1-2hr): Pipeline Depth by Phase section, truncation footnotes, cross-skill hints. P2 (defer): trials, patents, rename. Half a day total to production.
